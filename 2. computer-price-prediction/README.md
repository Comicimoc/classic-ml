# Анализ датасета компьютеров

## Описание
Репозиторий "**computer-price-prediction**" содержит исследовательский анализ данных (EDA) по объявлениям о продаже компьютеров/ноутбуков, а также реализацию моделей и их сравнение.
Датасет взят с kaggle.


## Цели проекта

- Сделать EDA-анализ
- Провести feature engineering
- Построить модели (LinearRegression, Lasso, Ridge, DecisionTree) и сравнить их


## Стэк технологий
- Python
- Scikit-learn, NumPy, Pandas, Matplotlib (Seaborn)
- PyCharm, Jupyter Notebook


## Результаты
1. Проведен EDA, в рамках которого отмечены следующие наблюдения:
   - Имеются признаки с высокой кардинальностью: "model" и "cpu_model"
   - Пропуски отсутствуют
   - У целевого признака "price" наблюдается значительное количество выбросов. Однако эти значения   являются естественными для рынка (не технические ошибки), поэтому их удаление нецелесообразно, так как они содержат важную информацию о дорогом сегменте.
   - В рамках корреляционного анализа отмечено, что лучше всего с price коррелируют такие признаки: "ram_gb", "cpu_tier", "gpu_tier", "cpu_cores"

2. Так как в датасете есть выбросы, которые нельзя удалять, в качестве основной метрики выбрана MAE, но для эксперимента также рассматривались такие метрики как RMSE, MAPE, R^2

3. Проведен feature engineering, в рамках которого:
   - Признак "resolution" с форматом "width x height" преобразован в признак "total_pixels" с форматом "int(width) * int(height)"
   - Удалены столбцы "model", "cpu_model", имеющие высокую кардинальность
   - Удалены мультиколлениарные столбцы "cpu_threads", "cpu_boost_ghz", "cpu_tier", "battery_wh", "charger_watts", "psu_watts"

4. Данные были преобразованы для дальнейшего моделирования:
   - Числовые признаки стандатизированы через StandardScaler(), а категориальные закодированы через OneHotEncoder()
   - Был создан дополнительно прологорифмированный target (y_log). Впоследствии модель строилась двух видов: по датасету с исходным распределнием таргета и прологорифмированным таргетом.

5. Реализованы модели LinearRegression, Lasso, Ridge, DecisionTreeRegressor
   - Лучше оказалась Ridge модель (по всем рассматриваемым метрикам: R^2 = 0.874863, RMSE = 203.278071, MAE = 143.604355, MAPE = 0.076905). Вариант с использованием прологорифмированного таргета не показал улучшений, а наоборот - ухудшил значение метрик (R^2 = 0.868461, RMSE = 208.413060, MAE = 146.541963, MAPE = 0.077459)
   - Для Lasso, Ridge и DecisionTree проведен подбор оптимальных гиперпараметров. Лучший alpha для Lasso - 0.001, Лучший alpha для Ridge - 5.0. Лучшие параметры DecisionTree: {'model__max_depth': 15, 'model__min_samples_leaf': 10, 'model__min_samples_split': 2}
   - После подроба оптимальных гиперпараметров проведено финальное сравнение моделей. Лучше снова оказалась Ridge (c best alpha = 5.0): R^2 = 0.874874, RMSE = 203.269233, MAE = 143.597145, MAPE = 0.076899. Но если сравнивать Ridge(c best alpha = 5.0) c Ridge(c alpha = 1.0), то в целом значительного прироста нет.
